# Multi-Input Nodes: Enabling True Fan-In Composition

## Context

Junction Box needs ComfyUI-style multi-input nodes that can accept multiple incoming edges on a single port and intelligently combine/merge the data. This is the **core missing piece** for multi-source data pipelines.

## Current Problem

When multiple nodes are wired to the same input port, only ONE value is used — the rest are silently lost. This was partially addressed with validation, but the real need is: **nodes that can genuinely handle multiple inputs**.

**The architecture gap:**
- Execution engine passes ONE value per param via env var (line 119-125 in `execute.ts`)
- When multiple edges target the same port, only the last one wins
- `append`, `merge`, `join`, `table-concat` are documented as multi-input but don't work because they receive a single NUON string, not an array

## Solution

### 1. Execution Engine: Pass Arrays for Multi-Edge Params

**File:** `server/execute.ts`

When multiple edges wire to the same wirable param, collect ALL values into a JSON array and pass via env var. The primitive receives an array it can iterate over.

```typescript
// Current behavior (line 119-125): single value
const paramEdge = graph.edges.find(e => e.to === nodeId && e.to_port === paramName)
if (paramEdge) {
  paramEnv[envKey] = outputs.get(paramEdge.from) ?? 'null'
}

// New behavior: collect ALL edges for wirable params into array
const paramEdges = graph.edges.filter(e => e.to === nodeId && e.to_port === paramName)
if (paramEdges.length > 0) {
  // For wirable params, collect all values into array
  const values = paramEdges.map(e => outputs.get(e.from) ?? 'null')
  paramEnv[envKey] = JSON.stringify(values)  // Pass array as JSON
}
```

### 2. Primitives: Handle Arrays for Multi-Input

Update primitives that have wirable params to properly handle arrays when multiple edges were wired.

**Pattern for list combining (`append`):**
```nushell
# If $items is an array (multi-input), concat all; if single NUON, parse normally
let extra = if ($items | from json | describe) == "list" {
    $items | from json
} else {
    ($items | from nuon)
}
$input | append $extra
```

**Pattern for record merging (`merge`):**
```nushell
let overlay = if ($with | from json | describe) == "list" {
    ($with | from json | reduce {|item, acc| $acc | merge $item })
} else {
    ($with | from nuon)
}
$input | merge $overlay
```

**Pattern for table concat (`table-concat`):**
```nushell
let more_tables = if ($more | from json | describe) == "list" {
    $more | from json
} else {
    ($more | from nuon)
}
$more_tables | reduce {|t, acc| $acc | append $t }
```

### 3. Key Multi-Input Nodes to Fix

Update these primitives to properly handle arrays when multiple edges were wired:

| Node | Current Behavior | Desired Behavior |
|------|------------------|------------------|
| `append` | Takes single NUON list, appends to input | Iterates array, appends all elements |
| `merge` | Takes single NUON record | Reduces array, merges all records into input |
| `join` | Takes single `--right` table | Performs multiple SQL joins or unions |
| `table-concat` | Takes single `--more` table | Reduces array, unions all tables |
| `math` | Takes single `--operand` | Chains operation across all operands |
| `str-concat` | Takes prefix/suffix strings | Applies prefix/suffix to multiple inputs |

### 4. Validation Update

**File:** `server/validate.ts`

**Remove** the `multiple_inputs` error entirely OR convert to a **warning**. With automatic arrays:
- Multiple edges to wirable params are now the EXPECTED behavior
- Primitives handle collecting values into arrays
- No need to distinguish "multi-input capable" nodes

```typescript
// REMOVE this check entirely — multiple edges to wirable params are valid
// if (count > 1 && !supportsMultiInput) { errors.push(...) }
```

### 5. New Multi-Input Nodes

Consider adding a purpose-built node:

- **`concat`** — Generic concatenator: accepts any mix of lists/tables, outputs single combined list
- **`combine-json`** — JSON-aware merge: deep merges objects, concatenates arrays
