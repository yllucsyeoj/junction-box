# Plan: Per-Node File Structure + Core Node Test Coverage

## Decisions

- **File structure**: Per-node files (each node = .nu + .test.ts + .example.json)
- **Test scope**: Core nodes only (transform, input, compute, datetime, logic ≈ 75 nodes)
- Extensions get smoke tests (just run, verify no crash)
- 117 total nodes, ~75 get full coverage, ~56 get smoke tests

## Phase 1: Restructure Core Primitives (primitives.nu → per-node files)

### Step 1: Create directory structure
```bash
mkdir -p primitives/input primitives/datetime primitives/compute primitives/logic primitives/output primitives/transform
```

### Step 2: Split primitives.nu into per-node files

Each file:
```nu
# primitives/transform/math.nu
export def "prim-math" [
    --op: string = "+"    # Operation: + - * / %
    --operand: string = "0"
]: number -> number {
    let a = $in
    let b = ($operand | into number)
    match $op {
        "+" => { $a + $b }
        "-" => { $a - $b }
        "*" => { $a * $b }
        "/" => { $a / $b }
        "%" => { $a mod $b }
    }
}
```

### Step 3: Create mod.nu per directory
```nu
# primitives/transform/mod.nu
use math.nu *
use filter.nu *
use select.nu *
# ... all transform nodes
```

### Step 4: Main primitives.nu becomes
```nu
# primitives.nu - thin wrapper that loads all modules
use primitives/input/mod.nu *
use primitives/datetime/mod.nu *
use primitives/compute/mod.nu *
use primitives/logic/mod.nu *
use primitives/output/mod.nu *
use primitives/transform/mod.nu *
```

Same pattern for extensions: `extensions/hn/mod.nu`, `extensions/market/mod.nu`, etc.

## Implementation Order

### Phase 1: Create test files first (low risk, high value)
1. Create `tests/nodes/transform/` directory
2. Create test files for each of 75 core nodes (can generate from spec)
3. Create `tests/smoke.test.ts` for extension nodes
4. Run tests, fix failures
5. **Commit: "Add comprehensive node tests"**

### Phase 2: Restructure files (higher risk, do after tests pass)
6. Create `primitives/input/`, `primitives/transform/`, etc. directories
7. Move node definitions from primitives.nu to per-node files
8. Create mod.nu files
9. Update primitives.nu to use modular structure
10. Update server/spec.ts to load from new structure
11. Run tests to verify nothing broke
12. **Commit: "Restructure primitives into per-node files"**

The phased approach means:
- If restructure introduces bugs, existing passing tests will catch them
- We get value (test coverage) early, restructuring later
- Can abort restructure if too complex
