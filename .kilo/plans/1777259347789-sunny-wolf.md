# Plan: Build 4 Data Source Extensions

## Context
`next.md` lists data sources to add. Already built: Reddit, RSS, HN, Fear & Greed, Wikipedia, Options flow. User wants: GitHub (2) тЖТ CoinGecko (3) тЖТ BLS (4) тЖТ FRED (1).

## Order & Rationale
1. **GitHub** тАФ public repo metrics (stars, commits, contributors), no auth, rate-limited
2. **CoinGecko** тАФ crypto market cap/dominance, free tier (30/min, 10k/month)
3. **BLS** тАФ sector employment/wages, now requires free API key
4. **FRED** тАФ macro indicators, requires free API key

## Extension 1: GitHub (`extensions/github.nu`)

**API**: `GET https://api.github.com/repos/{owner}/{repo}` тАФ public, no auth (60 req/hr unauthed)

**Nodes**:
| Node | Description |
|------|-------------|
| `github_repo` | Fetch repo metadata: stars, forks, watchers, language, description, topics, created_at, pushed_at, open_issues |
| `github_contributors` | Top contributors with commit counts via `/contributors` endpoint |
| `github_commits` | Recent commit history via `/commits` endpoint |

**Key fields**:
- `github_repo`: name, full_name, description, language, stargazers_count, forks_count, watchers_count, open_issues, topics, created_at, pushed_at, url
- `github_contributors`: login, contributions, avatar_url
- `github_commits`: sha, author, message, date

**Notes**:
- Stats endpoints (`/stats/contributors`, `/stats/participation`) require auth for high-commit repos (>10k)
- Use `try/catch` wrapping on all field accesses for resilience

## Extension 2: CoinGecko (`extensions/coingecko.nu`)

**API**: `https://api.coingecko.com/api/v3/` тАФ free tier, 30 calls/min, 10k calls/month

**Nodes**:
| Node | Description |
|------|-------------|
| `coingecko_global` | Global crypto market cap, BTC dominance, altcoin season |
| `coingecko_simple` | Price/market cap/volume for specific coin IDs |
| `coingecko_markets` | List coins with market data (price, change%, market cap) |

**Endpoints**:
- `GET /global` тЖТ total market cap, BTC dominance
- `GET /simple/price?ids=bitcoin&vs_currencies=usd&include_market_cap=true` тЖТ single coin price
- `GET /coins/markets?vs_currency=usd&order=market_cap_desc&per_page=20` тЖТ top coins

**Key fields**:
- `coingecko_global`: total_market_cap, btc_dominance, altcoin_count, active_cryptocurrencies
- `coingecko_simple`: price, market_cap, 24h_change
- `coingecko_markets`: symbol, name, current_price, market_cap, price_change_percentage_24h, total_volume

## Extension 3: BLS (`extensions/bls.nu`)

**API**: `https://api.bls.gov/publicAPI/v2/timeseries/data/` тАФ **requires free API key** (register at data.bls.gov/registrationEngine). 25 queries/day free without key, 500/day with key.

**Nodes**:
| Node | Description |
|------|-------------|
| `bls_series` | Fetch one or more BLS series by ID (e.g. CPI, unemployment) |
| `bls_presets` | Popular preset series: CPI, Unemployment, Employment, Wages, PPI |

**Common Series IDs**:
| Indicator | Series ID |
|-----------|-----------|
| CPI-U | `CUUR0000SA0` |
| Unemployment Rate | `LNS14000000` |
| Total Employment | `CES0000000001` |
| Avg Hourly Earnings | `CES0500000003` |
| PPI | `WPSFD4` |

**Request format**: POST JSON with series_ids array and optional API key

**Key fields**:
- `bls_series`: series_id, name, date, value
- Returns table with date/value pairs

**Notes**:
- BLS key goes in `.env` as `BLS_API_KEY` тАФ fallback to env var
- Max 50 series per request, 20 years of data
- Try series search endpoint if user doesn't know ID

## Extension 4: FRED (`extensions/fred.nu`)

**API**: `https://api.stlouisfed.org/fred/series/observations/` тАФ **requires free API key** (register at fred.stlouisfed.org/docs/api/api_key.html). No explicit rate limit stated.

**Nodes**:
| Node | Description |
|------|-------------|
| `fred_series` | Fetch observations for any FRED series ID |
| `fred_search` | Search for series by keyword |

**Common Series IDs**:
| Indicator | Series ID |
|-----------|-----------|
| GDP | `GDPC1` |
| CPI | `CPALTT01USM659S` |
| Federal Funds Rate | `FEDFUNDS` |
| 10Y Yield | `DGS10` |
| M2 Money Supply | `M2SL` |
| Unemployment | `UNRATE` |
| Industrial Production | `INDPRO` |
| Retail Sales | `RRSFS` |

**Request format**: `GET /fred/series/observations?series_id=XXX&api_key=KEY&file_type=json`

**Key fields**:
- `fred_series`: date, value (with NUON string conversion for numeric values)
- `fred_search`: series_id, title, observations_start, observations_end

**Notes**:
- FRED key goes in `.env` as `FRED_API_KEY` тАФ fallback to env var
- Values are sometimes "." for missing data тЖТ filter those out
- Units parameter can return percent change, etc. тАФ keep default "lin" for raw values
