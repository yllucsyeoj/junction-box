# Bug Fix Plan: F-1 (rss-feed) and F-2 (append)

## Summary

Both bugs are **completely broken** nodes that return runtime errors:
- **F-1**: `rss-feed` → `"Data cannot be accessed with a cell path (expected: nothing)"`
- **F-2**: `append` → `"Eval block failed with pipeline input"`

---

## F-1: `rss-feed` Bug

### Root Cause
In `extensions/rss.nu` line 54:
```nu
let doc = (http get -H {User-Agent: $RSS_UA} $url_val)
```

Nushell 0.111's `http get` returns **raw text** (string), not parsed XML. The code then tries to access `| get tag` and `| get content` on the string, which fails.

### Fix
Add `| from xml` after `http get`:
```nu
let doc = (http get -H {User-Agent: $RSS_UA} $url_val | from xml)
```

### Verification
```bash
curl -s -X POST http://127.0.0.1:3001/exec -H "Content-Type: application/json" -d '{
  "nodes": [
    {"id": "rss", "type": "rss-feed", "params": {"url": "https://news.ycombinator.com/rss"}},
    {"id": "out", "type": "return", "params": {}}
  ],
  "edges": [
    {"from": "rss", "from_port": "output", "to": "out", "to_port": "input"}
  ]
}'
# Expected: result is a table with title, link, published, summary, author columns
```

## F-2: `append` Bug

### Root Cause
In `primitives.nu` lines 805-806:
```nu
let parsed = if (($items | from json | describe) | str starts-with 'list') {
    ($items | from json | each {|v| $v | from nuon} | flatten)
} else {
    ($items | from nuon)
}
```

**Bug**: When `items = "[1, 2, 3]"`:
1. `$items | from json` → `[1, 2, 3]` (list of **ints**, not strings)
2. `each {|v| $v | from nuon}` → pipes each int to `from nuon`, which only accepts strings
3. Error: `"only string input data is supported"`

The same issue exists for `["a", "b"]` - `from json` returns a list of strings, but `from nuon` expects quoted NUON strings like `"\"hello\""`.

### Fix
Simplify to parse as NUON directly without the broken `each` pattern:
```nu
export def "prim-append" [
    --items: string = "[]"           # NUON list or value to append
]: any -> list {
    let input = $in
    let parsed = ($items | from nuon)
    let desc = ($parsed | describe)
    if ($desc | str starts-with 'list') or ($desc | str starts-with 'table') {
        $input | append $parsed
    } else {
        $input | append [$parsed]
    }
}
```

This works because:
- `"[1, 2, 3]" | from nuon` → `[1, 2, 3]` (list of ints)
- `"\"hello\"" | from nuon` → `"hello"` (string)
- `"[4, 5]" | from nuon` → `[4, 5]` (list of ints)
- Works for any NUON literal (lists, records, strings, numbers, bools)

### Verification
```bash
curl -s -X POST http://127.0.0.1:3001/exec -H "Content-Type: application/json" -d '{
  "nodes": [
    {"id": "src", "type": "const", "params": {"value": "[1, 2, 3]"}},
    {"id": "extra", "type": "const", "params": {"value": "[4, 5]"}},
    {"id": "op", "type": "append", "params": {"items": "[]"}},
    {"id": "out", "type": "return", "params": {}}
  ],
  "edges": [
    {"from": "src", "from_port": "output", "to": "op", "to_port": "input"},
    {"from": "extra", "from_port": "output", "to": "op", "to_port": "items"},
    {"from": "op", "from_port": "output", "to": "out", "to_port": "input"}
  ]
}'
# Expected: result: [1, 2, 3, 4, 5]
```
