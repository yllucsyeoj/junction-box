## Task 3: Fix `str-concat` whitespace stripping (F3)

**Files:**
- Modify: `primitives.nu:107` (agent_hint — WRONG guidance about NUON quoting)
- Modify: `primitives.nu:729-746` (prim-str-concat implementation)

The root issue: `prim-str-concat` calls `$suffix | from json` on the raw param value. When `$suffix` is a static string like ` World`, `from json` strips or mangles it. The fix: use `try { $val | from json } catch { $val }` so static strings fall through cleanly. JSON-parsed wired inputs (which ARE valid JSON strings like `"hello"`) still work via the try path.

- [ ] **Step 1: Fix `prim-str-concat` implementation**

In `primitives.nu` lines 729-746:

```nushell
# Concatenate strings — prepend prefix and/or append suffix
export def "prim-str-concat" [
    --prefix: string = ""            # String to prepend (or wire an edge to this port)
    --suffix: string = ""            # String to append  (or wire an edge to this port)
]: string -> string {
    # Params arrive as raw strings for static values, or JSON-encoded strings for wired inputs.
    # Try JSON parsing first (handles wired case); fall back to the raw string (handles static case).
    let prefix_val = try { $prefix | from json } catch { $prefix }
    let suffix_val = try { $suffix | from json } catch { $suffix }
    $"($prefix_val)($in)($suffix_val)"
}
```

- [ ] **Step 2: Update the agent_hint for `str_concat` in PRIMITIVE_META (line 107)**

The current hint says "do NOT use NUON quoting" but the fix means you don't need to worry about this at all:

```nushell
# BEFORE
    str_concat:    {category: "compute",   color: "#eab308", wirable: ["prefix", "suffix"], agent_hint: "Concatenate strings: prepend --prefix and/or append --suffix. Both ports are wirable. Param values are plain strings — do NOT use NUON quoting (set prefix to label= not \\\"label=\\\").", param_options: {}}

# AFTER
    str_concat:    {category: "compute",   color: "#eab308", wirable: ["prefix", "suffix"], agent_hint: "Concatenate strings: prepend --prefix and/or append --suffix. Both ports are wirable. Param values are plain strings — whitespace is preserved.", param_options: {}}
```

**Files:**

The problem: success, validation-error, and runtime-error responses have different sets of keys. Fix: always include `fatal`, `skipped`, `outputs` in all responses (with empty/null values when not relevant). The `?outputs=none` flag still controls whether `outputs` is populated.
