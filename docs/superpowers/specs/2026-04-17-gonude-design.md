# GoNude: Nu Visual Pipeline System — Design Spec

## Context

Agents and humans both struggle with the verbosity and surface area of general-purpose code when
building data pipelines. Visual patching languages (Max/MSP, ComfyUI, Node-RED) solve this by
constraining composition to a small library of typed primitives — reducing errors, enabling fast
iteration, and producing a compact, serializable representation of the pipeline.

This project builds the Nu equivalent: a visual node canvas where nodes are Nu custom commands,
edges are typed data flows, and the saved artifact is a NUON file that can be run, scheduled,
committed to git, or generated by an LLM directly.

---

## Architecture

Four components with clean boundaries:

| Component | Technology | Responsibility |
|---|---|---|
| `primitives.nu` | Nushell module | All primitive commands — source of truth for node types |
| `scripts/introspect.nu` | Nushell | Introspects `primitives.nu` + extensions → JSON spec |
| `server/` | Bun + Hono | Serves frontend, exposes `/nodes` and `/run` SSE endpoint |
| `frontend/` | React + Vite + React Flow | Canvas, node library, config panel, run log |

**Startup sequence:**
1. Bun server starts
2. Calls `nu scripts/introspect.nu` → outputs JSON node spec
3. Caches spec in memory, serves at `GET /nodes`
4. Serves frontend at `GET /`

**Runtime sequence:**
1. Frontend fetches `/nodes` → renders library panel
2. User builds pipeline on canvas, clicks Run
3. Frontend POSTs NUON to `POST /run` as an SSE stream
4. Server toposorts graph, executes node-by-node, streams status events
5. Frontend updates node spinners/highlights, populates run log

## NUON Graph Format

The canonical artifact — what is saved, loaded, and executed.

```nuon
{
    nodes: [
        {id: "n1", type: "fetch", position: {x: 100, y: 200}, params: {url: "https://api.example.com/data"}}
        {id: "n2", type: "filter", position: {x: 350, y: 200}, params: {column: "age", op: ">", value: "30"}}
        {id: "n3", type: "display", position: {x: 600, y: 200}, params: {}}
    ]
    edges: [
        {id: "e1", from: "n1", from_port: "output", to: "n2", to_port: "input"}
        {id: "e2", from: "n2", from_port: "output", to: "n3", to_port: "input"}
    ]
}
```

Key decisions:
- `position` stored so canvas restores exactly on drag-to-load
- `params` holds UI-configured values; incoming edge value takes precedence at runtime
- `from_port`/`to_port` support multi-port nodes
- Valid NUON — `open patch.nuon | from nuon` works in Nu directly

## Node Spec (Introspection)

Each primitive uses the `prim-` prefix. Metadata comes from `PRIMITIVE_META` const in each module.

**Node spec JSON shape:**
```json
{
    "name": "filter",
    "category": "transform",
    "color": "#3b82f6",
    "agent_hint": "Filter table rows: pick column, op (>, <, ==, !=, contains), and value",
    "description": "Filter table rows by column comparison",
    "ports": {
        "inputs": [{"name": "input", "type": "table"}],
        "outputs": [{"name": "output", "type": "table"}]
    },
    "params": [
        {"name": "column", "type": "string", "required": true, "description": "Column name to filter on"},
        {"name": "op", "type": "string", "required": true, "description": "Operator: >, <, ==, !=, contains"},
        {"name": "value", "type": "string", "required": true, "description": "Comparison value"}
    ]
}
```

## Extension System

Extensions live in `extensions/` — drop a `.nu` file in and restart the server. Never edit `primitives.nu` for domain-specific nodes.

**Convention:**
- Commands: `prim-<extension>-<action>` (e.g. `prim-nocodb-fetch-table`)
- Metadata export: `<EXTENSION>_PRIMITIVE_META` record (optional — defaults auto-derived from name)
- Category auto-derived from the extension segment of the command name

`introspect.nu` uses `overlay use` (runtime dynamic loading) to pick up all extension files.

## Execution Engine

**Toposort:** Kahn's algorithm. Cycles rejected before execution starts.

**Per-node:** `nu -c "use primitives.nu *; <nuon_input> | prim-<type> <args> | to nuon"`. Previous node output passed as stdin, parsed with `from nuon`. Edge-connected params passed via env vars.

**SSE events:** `{node_id, status: running|done|error, output?, error?}` + `{status: complete}`.

---

## Execution Engine

**Toposort:** Kahn's algorithm on edges. Cycles detected and rejected before execution starts.

**Per-node execution:** `nu -c "use primitives.nu *; <input> | prim-<type> <args> | to nuon"`.
Output piped as NUON between nodes. Edge-connected params via env vars.

**SSE events:** `{node_id, status, output?, error?}` streamed as each node completes.

## Frontend Layout

```
┌─────────────┬──────────────────────────────┬───────────────┐
│ Node Library│         Canvas               │  Node Config  │
│             │     (React Flow)             │               │
│ [Input]     │                              │  url: ______  │
│  fetch      │  [fetch]──>[filter]──>[display]             │
│  const      │                              │               │
│ [Transform] │                              │               │
│  filter     │                              │               │
└─────────────┴──────────────────────────────┴───────────────┘
│  [▶ Run]  [↓ Save NUON]              Run Log               │
└────────────────────────────────────────────────────────────┘
```

Port colors: `table`=blue, `string`=green, `number`=yellow, `any`=grey.

| Category | Primitives |

| input | fetch, const, env, file-in |
| transform | filter, map, select, sort |
| output | display, file-out, return |
| external | llm |
| compute | math, string-op, type-cast |
